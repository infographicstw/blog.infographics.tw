六個訣竅，改進你的視覺化作品

Posted 7 8 月, 2015 by infographics.tw under 心法

資料視覺化並不難做，但做出一個優秀的視覺化作品卻是一門大學問。為了幫助大家做出更棒的視覺化作品，我們整理了六條關於視覺化相當基礎卻很容易被忽略的問題，只要能徹底的實踐，一定能做出好作品來。

一、慎重挑選你的字體

So you need a typeface – by julian hansen

很多字體都有其使用的時機，例如古印體常常用在恐怖情節，而漫畫體則應用在輕鬆的內容。而字體的粗細也是一個考量的點，在不同的圖像中，很可能因為背景或是視覺化元素的不同讓字型變化產生適用性的問題。

在設計階段，記得參考其它類似屬性的作品，紀錄他們使用的字體，從中挑選一些在你的作品中測試。有必要時，也可以尋找相似的線上字型來測試比較，但到最後完稿之前，避免在同一份作品中使用太多的字型，基本上以同個作品內不超過 3 種字形為佳。

 

二、使用恰當的顏色

顏色往往是視覺化作品的一個大賣點，但也往往為人所忽略，結果搞砸了一個原本的好作品。要挑選顏色，可以使用色盤產生器，例如上圖為 loading.io 提供的 「Infinite Palettes」無限色盤，可以透過關鍵字或指定色票來提供色盤產生的參考值；或是也可以到 ColourLovers 、 HailPixel 等色盤分享網站尋找靈感，然後利用 Color Scheme Designer 這類的工具做適當的調整。

有些網站例如 Color Scheme Designer 或是 Contrast-A 提供相當方便的測試工具，包含色盲、對比等等，避免你的配色令人難以閱讀，或是使色盲的網友無法正確的分辨你圖表中的元素。網頁親和力就從配色做起，千萬不要在這部份輕忽了！

另外，loading.io Colors 同時還提供 d3.js 的 API  及套用到圖表上的預覽 ( 包括 chart.js 的長條圖與 d3.js 的泡泡圖 )  ，例如下圖即為迪士尼「冰雪奇緣」色盤的圖表預覽：

三、留白的藝術

空白，包括邊框、字距、行距等等，對一個清楚明瞭的作品來說是相當重要的，因為人腦在辨識圖形時並不是以有沒有東西做為依據，而是會因圖形周圍的元素暗示而在認知中出現假想的物件。其中一個相當重要的概念就是圖表的節奏：透過不同間隔的空白，你就能暗示讀者你如何將圖表的內容分類。

如果覺得空白很難掌握，那沒關係，記住一個最簡單的概念即可：較少的留白拉近元素的關係，而較大的留白推開元素的關係。

另外值得一提的是，在製作視覺化的時候，要注意是否有因為空白使用不當而產生了視覺上的幻影，例如相當有名的「Grid Illusion」，在黑色方格的角落彷彿看到了灰色的圓點：

 

 

四、利用對齊引導讀者的理解

與留白同樣重要，對齊也暗示著讀者各種元素間的關係。一連串對齊的圖示中有一個突出的圖示 ─ 也許是選中，也許是提示，端看你如何表現其概念。對齊的內容很容易就讓人感到一種規律，更容易遵循閱讀下去，也因此你可以省下更多解說的空間及工夫。

一個最簡單的例子，底部未對齊的長條圖便讓人很難比較大小。有些圖表會在圖上各處呈現長條，立即就會遭遇這個問題，所以當你碰到這類的需求時，務必要考量清楚對齊對圖表的衝擊。

 

五、善用各式視覺元素

The Noun Project: http://thenounproject.com/

有時穿插圖示效果比起文字描述會好上許多。例如，比較 Android 與 iPhone 時，與其寫出文字，不妨置入小綠人與蘋果來表現雙方陣營。抽象的圖形也很適合利用，例如性別、方向、情緒、物品等等。重點是，在使用各種不同圖形時，除非完完全全是自己繪製的，否則來源可能大不相同。確定各個不同的元素風格都類似，避免過於零碎的圖表設計，是挑選圖形時的一個重大考量。

網路上有各式各樣的圖形資源，但說到抽象圖示就還是不得不提我們先前介紹過的 The Noun Project ，在這個網站你可以透過很快的找到你想要的圖形，重點是數量相當的多，你可以慢慢尋找適合的，同時作品都有標示授權方式，大多是 CC0 或者 CC-BY ，你可以依照符合你的需求來使用這些圖示。

 

六、事先考量閱讀的媒介

2015 年的今天已經有相當高的手機普及率，同時還有平板電腦、桌上型電腦、筆記型電腦等等各種的瀏覽裝置。你的作品可能會出現在哪裡？這會影響到你的構圖比例、字體與圖示比例、資訊複雜程度以及互動的方式。這裡列出一些常見會犯的錯誤，讓大家可以很容易的避免：

  1. 靠滑鼠移至圖表上方獲得某些資訊 －要注意這樣的行為在觸控裝置上會造成困擾。
  2. 固定的字體大小 －有些在桌機上恰當的字體大小在手機上便會顯得過小。
  3. 固定的邊框留白 － 在小螢幕中，留白常會相對的大，使得設計整體比例變形。
  4. 未考慮手機直握的狀況 － 使用手機的情境大多是直握，這與桌機閱讀體驗截然不同。

前陣子我們也分享了一篇關於作品發表前的測試工具「視覺化作品發表前，別忘了用這些工具測試一遍！」一文，裡面有提到如何使用現成的工具快速在不同媒體上測試自己的作品，值得一看。
