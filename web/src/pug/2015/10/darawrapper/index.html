四個步驟作圖表 — 開源圖表工具 Datawrapper

Posted 21 10 月, 2015 by infographics.tw under 介紹

你曾經覺得需要一個量身打造的統計圖表工具嗎？線上圖表工具用過了百百種，真的要挑一個給公司、團隊的同事們使用，有時候還是綁手綁腳的。 Datawrapper 本身不僅是一個線上圖表工具，它的原始碼同時也一併釋出，如果需要客製調整，完全可以自己來！那麼，我們就先從 Datawrapper 的使用面上一起來看看他有什麼特色吧！

Datawrapper 的設計與其它圖表工具相仿，我們可以建立新圖表、匯入資料、最後匯出，也提供了嵌入的選項。不過，圖表製作流程設計為四個步驟，不需登入也可以製作圖表，以及恰到好處的圖表選項 ( 不像 amcharts 那般巨大而令人生畏 … ) ，都為 Datawrapper 帶來了一些獨特性。

連到首頁後點選選「+ New Chart」開始建立新圖表，我們可以看到如上的畫面；其中可以看到說好的四個步驟，包含了：

  1. 上傳資料 ( Upload Data )
  2. 檢查資料 ( Check & Describe )
  3. 視覺化 ( Visualize )
  4. 發表 ( Publish & Embed )

1. 上傳資料

Datawrapper 提供上傳或貼上CSV 檔的選項，讓我們可以使用我們自己的資料；如果手頭上沒有資料，也可以點選左方的「Select a sample dataset」來直接帶出一些示範用的資料。示範用的資料各有各適合的圖表類型，在下拉式選單中已經做好分組，可以參考著使用：

2. 檢查資料

資料上傳之後， Datawrapper 會將處理的結果預覽提供給我們參考，這時我們可以簡單瀏覽一下資料是否正確，同時對有問題的地方進行修改。然而 Datawrapper 畢竟不是像 Open Refine 這樣強大且專門的資料處理工具，這裡頂多就是轉置表格、修改數值或做資料列的格式調整：

基本上建議先將資料利用其它工具調整成想要的格式在進行上傳。

3. 視覺化

在這個階段，我們選擇想用的圖表，並且進行調整。免費用戶可以使用的圖表類型主要有四類，長條圖、圓餅圖、線圖、地圖，也可以使用表格表現；各種表格還包含了幾種不同的變形，比方說堆積或分組式長條圖、甜甜圈圖等等。

不同的圖表有不同的設定項，這裡不一一介紹，我們可以從下圖一窺長條圖的設定項：

除了基本的顏色與圖例設定外，你還可以看到像是資料來源跟網址的設定。一般在做資料視覺化 — 尤其是應用在新聞報導的時候 — 資料來源是相當重要的， Datawrapper 顯然將這個需求直接考慮了進去。為了方便敘事， Datawrapper 也提供強調單一元素的功能，搭配文字描述可以讓讀者更專注於主題上。

另外值得一提的是 Datawrapper 也提供選擇設計元素的功能，預設我們可以看到有「Datawrapper」與「Playfair」兩種可以選取，除了前面我們看到、類似 Google Charts 式的圖表即為  Datawrapper 預設樣式外，下圖為 Palyfair 的範例：

雖然這裡只有兩種設計樣式可以挑選，但 Datawrapper 有提供客製樣式的服務，供需要有獨特樣式的團隊使用。這是個收費服務，如果有需求大家可以參考看看。

4. 發表 ( Publish )

完成之後，Datawrapper 提供列印或網頁原始碼下載方式的輸出，使用原始碼下我們可以直接將圖表置入網站中，也可以另外利用 iframe 嵌入。

Datawrapper 也提供圖檔輸出以及線上嵌入圖表的選項，不過必需要將帳號升級至「Datawrapper Single」才能使用。下圖為發表圖表時有的選項：

 

結語

如同 RAW 一般， Datawrapper 提供了線上圖表製作工具以外，也一併提供了原始碼供大家下載使用；不過，兩者的設計邏輯截然不同； RAW 強調不經手資料、 Datawrapper 則提供登入、上傳、存檔等功能； RAW 產生了 PNG 、 SVG 等格式供我們下載， Datawrapper 則提供下載與嵌入功能。RAW 以提供特殊圖表為主， Datawrapper 則提供了常見的圖表類型。除此之外， Datawrapper 也提供各種客製化服務，例如針對團隊需求製作當地的地理向量地圖，或總地來說， Datawrapper 有著下列的特色：

  • 客製化服務，包含樣式、圖表等等
  • 開放原始碼、提供自架的選項

Datawrapper 所提供的不光是單純的圖表，其實背後還包含了完整的服務，從團隊經營的角度來看等於是外包了圖表平台架設，同時無論是嵌入、存檔等功能都便於團隊管理，也因此無論是直接利用 Datawrapper 提供的服務，或是取得原始碼以後自行改裝架設，都相當的方便。

因此，如果你們是一個編輯團隊，有圖表製作需求卻缺乏相關領域的專家，亦或者有工程師的配合但又不想重頭打造類似的平台，都可以嘗試看看 Datawrapper 喔！
